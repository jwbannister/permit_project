# Deliverable Chain: 270-10-1 Work Force Hub

## Purpose

This document maps the complete input-to-output chain for every deliverable produced during the Work Force Hub air quality permitting project. For each deliverable, we identify:
- **Inputs** — what data, documents, or prior work products fed into it
- **Transformation** — what analysis, calculation, or professional judgment was applied
- **Output** — the deliverable itself and its format
- **Dependencies** — which other deliverables had to exist first

This mapping is essential for designing an AI agent that can reproduce the workflow: the agent needs to know not just *what* to produce, but *in what order* and *from what*.

## Phase 1: Data Gathering and Source Characterization

### [1] Emissions Inventory Workbook

**File:** `00_EI/LN WFH EmisInv_LnPwr_r1_NDEP.xlsm`

**Inputs:**
| Input | Source | Example |
|-------|--------|---------|
| Equipment specifications | Client / manufacturer datasheets | Cummins 1000 kW T4 diesel engine specs |
| Fuel types and consumption rates | Client operating parameters | ULSD, LPG consumption rates |
| Operating hours (annual) | Client / regulatory assumptions | 500 hrs/yr for emergency generator |
| Emission factors (hierarchical) | NSPS/CFR standards, EPA AP-42, mass balance | 40 CFR 60 Subpart IIII Tier standards (diesel), AP-42 Ch. 3.3/1.5 (fallback), fuel sulfur mass balance (SO2) |
| GHG emission factors | EPA mandatory reporting rule | 40 CFR Part 98 |
| HAP speciation profiles | EPA speciation data | Diesel exhaust HAP fractions |
| Source coordinates | Site plan / surveyor data | UTM Zone 11N coordinates |
| Building dimensions | Site plan / engineering drawings | Heights, widths, footprints |
| Stack parameters | Equipment specs / engineering judgment | Stack height, diameter, exit velocity, temperature |
| Regulatory thresholds | Nevada Class II permit rules | PTE limits, significance levels |

**Transformation:**
The workbook is a 14-sheet Excel macro-enabled file that serves as the central calculation engine. Key sheets:

| Sheet | Function |
|-------|----------|
| **Proc** | Core emissions processing — 154 columns performing source-by-source emission calculations. Columns A-K hold source identification, L-AJ hold emission factors by pollutant, AK-EX hold calculated emission rates (lb/hr, ton/yr) for each pollutant and averaging period |
| **HAP&GHG** | Hazardous Air Pollutant and greenhouse gas calculations with speciation factors |
| **Tanks** | Liquid storage tank emission calculations (propane tank losses) |
| **Conv** | Unit conversion factors and constants used throughout |
| **INPUT_05_linepwr** | AERMOD input file generator — VBA macro reads from Proc sheet and writes `.INP` files with proper AERMOD syntax |
| **RE_grid** | Receptor grid coordinates (boundary, near-field, far-field) |
| **RE_bndry** | Boundary receptor coordinates at 25m spacing |
| **BPIP** | Building footprint vertices and heights for BPIPPRM downwash calculations |
| **Sources** | Source identification and location data |
| **Summary** | Rollup of emissions by source and pollutant |

**Outputs:**
- Calculated emission rates (lb/hr, ton/yr) for CO, NOx, PM10, PM2.5, SO2, HAPs, GHGs
- PTE (Potential to Emit) determination
- AERMOD-ready source parameters (location, stack dimensions, emission rates)
- Auto-generated AERMOD `.INP` files (via VBA macro)
- BPIPPRM input data

**Dependencies:** None — this is the foundational deliverable. Everything else flows from it.

**Revision history:** Archive folder shows iteration from generator option to line power option. The NDEP submission version (`_r1_NDEP`) is the final.

## Phase 2: Dispersion Modeling

### [2] AERMOD Input Files

**Files:** 45 `.INP` files in `01-Permitting/Modeling/External Distribution/To NDEP - February 2025 - Impact Analysis - linepwr/AERMOD Files - it05/`

**Inputs:**
| Input | Source |
|-------|--------|
| Source parameters | EI workbook Proc sheet (auto-generated by VBA macro) |
| Receptor grid coordinates | EI workbook RE_grid and RE_bndry sheets |
| Meteorological data file paths | Met Data directory (AERMET-processed .SFC and .PFL files) |
| Building downwash parameters | BPIPPRM output |
| Terrain elevations | NED 1/3-arc-second data (via AERMAP) |
| Modeling options | Regulatory defaults per EPA guidance |

**Transformation:**
The VBA macro on the INPUT_05_linepwr sheet auto-generates INP files by:
1. Reading source emission rates and stack parameters from the Proc sheet
2. Formatting them into AERMOD's five-section syntax (CO, SO, RE, ME, OU)
3. Writing one file per pollutant/averaging period/met year combination

**INP file structure:**
```
CO section — Control options (MODELOPT DFAULT, POLLUTID, AVERTIME, FLAGPOLE)
SO section — Source definitions (LOCATION, SRCPARAM, BUILDHGT, BUILDWID for each source)
RE section — Receptor definitions (GRIDCART for grids, DISCCART for boundary)
ME section — Meteorological file paths (.SFC surface, .PFL profile)
OU section — Output options (RECTABLE, MAXTABLE, PLOTFILE)
```

**Output structure — 9 pollutant/averaging scenarios x 5 met years = 45 runs:**

| Directory | Pollutant | Averaging | NAAQS Standard |
|-----------|-----------|-----------|----------------|
| CO_ST | CO | 1-hr, 8-hr | 40,000 / 10,000 ug/m3 |
| NOx_ST | NO2 | 1-hr | 188 ug/m3 (ARM2) |
| NOx_LT | NO2 | Annual | 100 ug/m3 |
| PM10_ST | PM10 | 24-hr | 150 ug/m3 |
| PM25_ST | PM2.5 | 24-hr | 35 ug/m3 |
| PM25_LT | PM2.5 | Annual | 12 ug/m3 |
| SO2_ST | SO2 | 1-hr | 196 ug/m3 |
| SO2_ST_1 | SO2 | 3-hr | 1,300 ug/m3 |
| SO2_LT | SO2 | Annual | 80 ug/m3 |

**Naming convention:** `{POLLUTANT}_{PERIOD}_{YY}.INP` — e.g., `CO_ST_17.INP` = CO short-term, 2017 met year.

**Dependencies:** Emissions Inventory Workbook [1]

### [3] AERMOD Model Runs

**Files:** 45 `.OUT` files (paired with each `.INP`), plus `.PLT` plot files

**Inputs:**
| Input | Source |
|-------|--------|
| AERMOD input files | Auto-generated `.INP` files [2] |
| AERMOD executable | `aermod.exe` v23132 |
| Meteorological data | AERMET-processed files (5 years: 2017-2021, Winnemucca surface / Elko upper air) |
| Terrain/receptor elevations | AERMAP-processed NED data |

**Transformation:**
- Batch files (`.bat`) execute AERMOD for each INP file
- AERMOD performs Gaussian plume dispersion calculations using hourly meteorology
- For NO2 scenarios, ARM2 (Ambient Ratio Method 2) is applied with ratio bounds 0.5-0.9
- Model produces maximum concentration predictions at each receptor for each averaging period

**Outputs:**
- `.OUT` files — Full model output including maximum concentrations, receptor-by-receptor results
- `.PLT` files — Plot files with concentration isopleths for mapping
- Console output with run statistics and warnings

**Dependencies:** AERMOD Input Files [2], Meteorological Data, BPIPPRM Output [3a]

### [3a] BPIPPRM Building Downwash

**Files:** BPIPPRM input/output in `01-Permitting/Modeling/Setup/`

**Inputs:**
| Input | Source |
|-------|--------|
| Building footprint vertices | EI workbook BPIP sheet |
| Building heights | Site plan / engineering drawings |
| Source locations relative to buildings | EI workbook Sources sheet |

**Transformation:**
BPIPPRM (Building Profile Input Program for PRIME) calculates direction-specific building dimensions that AERMOD uses to account for aerodynamic downwash effects when plumes interact with nearby structures.

**Outputs:**
- Building downwash parameters (BUILDHGT, BUILDWID arrays) incorporated into AERMOD INP files
- 36-direction building profiles for each source-building combination

**Dependencies:** Emissions Inventory Workbook [1]

### [4] Results Post-Processing

**Files:** `01-Permitting/Modeling/Results/LNC_Grab_Plots_v05_full_linepwr_20250214.xlsm`

**Inputs:**
| Input | Source |
|-------|--------|
| AERMOD .PLT files | Model run outputs [3] |
| NAAQS/significance thresholds | EPA and Nevada regulatory tables |
| Background concentrations | NDEP-provided or EPA monitoring data |

**Transformation:**
The Grab_Plots workbook (macro-enabled) reads AERMOD plot files and:
1. Extracts maximum concentrations across all receptors and met years
2. Compares each pollutant/averaging period result against the applicable NAAQS
3. Adds background concentrations for cumulative impact assessment
4. Determines compliance (project impact + background < NAAQS)
5. Generates summary tables for the impact report

**Revision history:** Six versions tracked (v01 through v05, plus v04-linepwr), corresponding to the iteration history.

**Outputs:**
- Maximum modeled concentration for each pollutant/averaging period
- Compliance determination (pass/fail against NAAQS)
- Summary tables formatted for the impact report
- Data for GIS impact maps

**Dependencies:** AERMOD Model Runs [3]

## Critical Path Analysis

The **critical path** through the deliverable chain is:

```
Client Data → EI Workbook → AERMOD INP Files → Model Runs → Results Processing → Impact Report → Application Assembly
```

This path determines the minimum project duration. Every other deliverable either branches off from the EI workbook (forms, narrative, GIS) or follows the application submission (RFI, compliance).

**Bottleneck:** The Emissions Inventory workbook is the single point through which nearly all project data flows. Its completeness and accuracy determine the quality of every downstream deliverable.

**Iteration risk:** The modeling phase (Iteration01 through Iteration05) shows that model runs often require refinement — changing source types, adjusting parameters, correcting errors. The project history shows 5 iterations over ~4 weeks, suggesting this is where the most technical judgment is exercised.

## AI Reproduction Assessment

Based on the deliverable chain mapping, an AI agent's ability to reproduce each component:

| Deliverable | Reproducibility | Key Challenge |
|-------------|----------------|---------------|
| Emissions Inventory | **High** | Selecting correct AP-42 factors; structuring 154-column calculation engine |
| AERMOD Input Files | **High** | Mechanical transformation from EI data; well-defined format |
| AERMOD Model Runs | **High** | Execution is deterministic given inputs; batch automation exists |
| BPIPPRM | **High** | Formulaic calculation from building geometry |
| Results Post-Processing | **High** | Tabular extraction and comparison against known thresholds |
| Application Forms | **High** | Mail-merge from structured data; NDEP templates are fixed |
| Technical Narrative | **Medium** | Requires regulatory knowledge and professional writing conventions |
| Impact Report | **Medium** | Requires modeling expertise to interpret results and frame compliance |
| GIS Figures | **Medium** | Requires spatial analysis skills; format conventions vary |
| Cover Letter | **High** | Standard professional format with project-specific details |
| RFI Response | **Low-Medium** | Requires interpreting ad hoc agency questions; judgment-intensive |
| Compliance Package | **Medium** | Requires parsing permit conditions into trackable obligations |

**Overall assessment:** The technical core (emissions calculations → modeling → results) is highly reproducible because it follows well-defined EPA methodologies with deterministic calculations. The professional communication layer (narratives, reports, RFI responses) requires more judgment but follows recognizable templates. The highest-risk element for AI reproduction is the **iterative modeling refinement** — knowing when results look wrong, diagnosing why, and deciding what to change.
